# Security and privacy reporting

Security and privacy reports are welcome. Prefer GitHub's private vulnerability
reporting feature for this repository when it is available. If it is not
available, contact the repository owner privately through their GitHub profile
before disclosing details.

Do not open a public issue containing:

- credentials, tokens, or private endpoints;
- participant data or identifying research records;
- a working exploit; or
- licensed third-party source.

If participant data have been committed or published, treat the event as a
privacy incident: stop further publication, preserve evidence privately, notify
the responsible research owner, and follow the applicable institutional process.

This archived prototype is not intended to process untrusted network input or
to provide production security guarantees.
